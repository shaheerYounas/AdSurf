01 Product Promise
Operating Model

What AdSurf Is Built To Do

AdSurf is an Amazon Ads AI Automation Control Center for sellers and agencies. It helps a user move from competitor keyword files and Amazon Ads performance reports into reviewed keyword lists, campaign plans, bulk sheet exports, monitoring insights, and approval-controlled recommendations.

Rules Calculate

Deterministic rule services score keywords, group campaigns, calculate metrics, and generate fallback recommendations.

AI Explains

AI can recommend, summarize, map, and explain evidence. AI output is bounded by structured schemas and validation.

Humans Approve

Human approval is required before customer-impacting actions, exports, recommendation decisions, or future live execution.

Who Should Use It

RoleBest UseTypical Actions
OwnerWorkspace governanceManage access, approvals, audit posture, product defaults.
AdminEnd-to-end operationsCreate products, upload files, generate plans, approve exports.
AnalystPreparation and reviewMap columns, score keywords, draft campaign plans, inspect evidence.
ApproverDecision controlApprove/reject keyword sets, campaign plans, recommendations, and exports.
ViewerRead-only visibilityInspect dashboards, reports, recommendations, monitoring, and audit evidence.
What the MVP does not do: it does not launch campaigns, change bids, pause campaigns, add negatives, lock campaigns, or call Amazon Ads live mutation APIs without explicit approval records. First execution mode is Amazon bulk sheet export.
02 Quick Start
First 30 Minutes

Fastest Safe Path From Setup To Review

  1. Open Dashboard. Confirm the workspace loads and use the launch checklist to see missing pieces.
  2. Create a product. Add product name, marketplace, currency, target ACOS, default daily budget, and default bid.
  3. Upload a research file. Use CSV, XLS, or XLSX. Confirm upload so the file enters the parsing queue.
  4. Open mapping. Generate a column profile, inspect sample values, and map required fields.
  5. Approve the mapping. The approved mapping snapshot becomes the input for scoring.
  6. Run keyword relevance scoring. Review approved/rejected candidates and override only with a reason.
  7. Create an approved keyword set. This freezes the reviewed terms for campaign planning.
  8. Generate and approve a campaign plan. Review Hero, Exact, Phrase, Broad, negatives, bids, and budget.
  9. Generate the bulk sheet export. Export only after an explicit approval note is saved.
  10. Import performance reports later. Use monitoring and recommendations to review post-launch account performance.
Dashboard gives the workspace overview, safety boundary, product continuation links, and launch checklist.

Mentor Layer
Reading The UI

How To Read Any AdSurf Page

Every page follows the same pattern: orientation at the top, action controls near the workflow area, evidence in panels or tables, and safety reminders before decisions. Once you learn this pattern, every page becomes easier.

1. Page header tells you the job: what this screen is for and what safety boundary applies.
2. Filter controls narrow the evidence. Start broad, then filter by status, source, priority, or type.
3. The table is the decision surface: evidence, reasoning, proposed action, status, and approval controls.
4. Safety strip repeats the rule: recommendation only, human approval, no live Amazon Ads change.

Where To Look First

Start at the page header, then look for the primary workflow control. Do not begin in the table until you know which workflow state you are in.

Where To Look Before Approving

Look for source file, mapping, metrics, reasoning, proposed action, status, and safety copy. If one is missing, approval is premature.

Decision Tree: “What should I do next?”
No product yet? Go to Products → New product.
Product exists but no file? Go to Product detail → Uploads.
File processed but no scoring? Go to Mapping → generate profile → approve mapping → run scoring.
Recommendations exist? Go to Recommendations → filter pending approval → approve/reject with notes.
Ready for export? Confirm plan approval and export approval before downloading the bulk sheet.

Micro-Steps
Mapping & Scoring Mentoring

Mapping: Teach AdSurf What The File Means

Mapping is where users often move too fast. Slow down here. A beautiful campaign plan from a wrong mapping is still wrong. Use the column profile like a teacher uses examples: source column name, normalized name, inferred type, and sample values.

Before You Click Save Mapping

1
Read sample values.

Column names can be misleading. Sample values reveal whether a column is a keyword, rank, sales metric, or campaign label.

2
Check required fields.

Search term and search volume are required for keyword scoring; performance fields are required for monitoring recommendations.

3
Approve only a valid snapshot.

Approval freezes the meaning of the source file for later scoring.

After Scoring Runs

4
Review effective status.

The effective status includes rule output plus any justified human override.

5
Override carefully.

Only override when you can explain why the rule result is not appropriate for this product.

6
Create keyword set.

The locked set is the reviewed input for campaign plans.

If You SeeIt MeansDo This
No profileAdSurf has not summarized columns yet.Generate profile before mapping.
Mapping invalidA required canonical field is missing or inconsistent.Fix field choices and save again.
Scoring unavailableMapping is not approved yet.Approve valid mapping snapshot.
No approved candidatesRules rejected all candidates or mapping is wrong.Inspect mapping and candidate evidence before overriding.

Scenarios
Pros, Cons, And Best Paths

Choose The Right Path For The Situation

Scenario 1: New product launch from competitor keyword research

Best path: Product → Upload competitor file → Mapping → Scoring → Approved keyword set → Campaign plan → Export.

Pros

Fast launch structure, deterministic scoring, clear approval trail, bulk sheet ready.

Cons

Competitor data may not reflect your conversion performance yet. Review relevance carefully.

Scenario 2: Running ads are spending but sales are weak

Best path: Upload Sponsored Products report → Monitoring → Recommendations → filter high/critical → review evidence → approve/reject.

Pros

Uses actual account evidence, supports negative keyword and pause-review decisions.

Cons

Bad date ranges or missing columns can create misleading recommendations.

Scenario 3: Agency wants account-level triage

Best path: Agents → Upload account report → Hybrid mode → Run analysis → inspect workflow → send recommendations to approval.

Pros

Best for many campaigns/products, gives trace events and agent-stage visibility.

Cons

Needs disciplined review because account-level uploads can span many entities.

Scenario 4: A user is unsure whether to approve

Best path: Do not approve yet. Check source file, mapping, evidence, rule/AI source, confidence, and business context. Reject or request better data if evidence is weak.

Pros

Protects account quality and audit trust.

Cons

Slower than blindly approving, but safer and aligned with the product mission.

06 Mapping
Columns To Canonical Fields

Map Columns, Score Keywords, And Review Candidates

The mapping page is the operational bridge between raw spreadsheets and rule-backed keyword decisions. Nothing should be scored until a user has reviewed and approved the mapping snapshot.

Mapping combines column profile generation, manual mapping, scoring, keyword review, campaign plan generation, and export approval controls.

Recipe: Turn A Parsed File Into Reviewed Keywords

  1. Open a processed upload and choose the mapping page.
  2. Generate or load the column profile.
  3. Inspect source column names, normalized names, inferred types, and sample values.
  4. Map the search term column, search volume column, and competitor rank columns.
  5. Save the manual mapping and fix validation errors.
  6. Approve the valid mapping snapshot.
  7. Run deterministic keyword relevance scoring.
  8. Filter candidates by status, score, and override state.
  9. Approve or reject candidates. Manual overrides require a reason.
  10. Create a locked approved keyword set snapshot.

Scoring And Override Rules

ConceptHow It WorksWhy It Matters
Deterministic scoringRules assign relevance status and score from mapped source fields.Reliable fallback and auditable decisions.
Rejected scoresScores 0, 1, and 2 are rejected by default.Prevents weak terms from entering campaign plans.
Manual overrideUser can approve or reject with a required reason.Captures expert judgment without hiding rule output.
Approved keyword setFrozen snapshot of effective approved candidates.Campaign plans must use reviewed inputs, not mutable draft rows.
07 Campaigns & Exports
Bulk Sheet First

Generate Campaign Plans And Bulk Exports

Campaign generation turns locked approved keyword sets into Sponsored Products manual campaign structures. The plan must be approved before export. The export must be explicitly approved before download.

Campaign Structure

  • Hero campaign from strongest keyword.
  • Exact, Phrase, and Broad group campaigns.
  • Negative Exact and Negative Phrase structures from rules.
  • Default bid and budget from product profile.

Export Boundary

  • Bulk sheet export is MVP execution mode.
  • Export generation requires approval evidence.
  • Downloaded files are manually uploaded to Amazon Ads.
  • No live Amazon Ads mutation occurs.
Approved Keyword CountCampaign BehaviorRecovery If Blocked

12 Cookbook Recipes
Repeatable Workflows

Practical Recipes

Recipe A: Launch From Competitor Keyword Research

  1. Create a product profile with default bid, budget, marketplace, currency, and target ACOS.
  2. Upload the competitor keyword file on the product Uploads page.
  3. Confirm upload and wait for processed status.
  4. Generate a column profile and map required keyword fields.
  5. Approve mapping, run scoring, review candidates, and override with reasons where needed.
  6. Create an approved keyword set and generate a campaign plan.
  7. Review Hero, groups, negatives, bids, and budget.
  8. Approve plan, approve export, download the bulk sheet, and manually upload it to Amazon Ads.

Recipe B: Review Running Ads After 14 Days

  1. Export a Sponsored Products Search Term report from Amazon Ads.
  2. Upload it to the correct product and wait for processed status.
  3. Open Monitoring and select the processed report upload.
  4. Create the monitoring import and run local processing in development if needed.
  5. Open Recommendations and filter by pending approval, priority, or type.
  6. Approve or reject each recommendation only after reviewing metric evidence and reasoning.

Recipe C: Agency Account-Level Triage

  1. Open Agents and upload an account-level Amazon Ads report or bulk sheet.
  2. Choose Hybrid mode for AI-assisted reasoning with deterministic fallback.
  3. Run analysis and inspect detection, product resolution, metric analysis, and recommendation stages.
  4. Use the Agent Inspector to review input, output, permissions, trace, and safety.
  5. Send actionable recommendations into the approval queue.
  6. Review decisions with notes. Do not treat approval as live account execution.
