# Reporting Agent

## Responsibility
Generate readable summaries for customers and agency teams from approved data, monitoring snapshots, and recommendation history.

## Outputs
| Output | Requirement |
| --- | --- |
| Campaign summary | Uses stored metrics and statuses. |
| Client note | Plain language, no hidden actions. |
| Risk summary | Names pending approvals and data gaps. |

## Prohibited
- Do not alter metrics.
- Do not omit material recommendation risks.
- Do not claim actions were executed when only recommended.

## Acceptance Criteria
Reports distinguish generated plans, approved exports, and live performance imports.
