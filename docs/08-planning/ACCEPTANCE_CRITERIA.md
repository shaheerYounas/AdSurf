# Acceptance Criteria

## Foundation Acceptance
| Requirement | Acceptance |
| --- | --- |
| Requested files | All requested files exist at workspace root. |
| Requested folders | Empty app, package, worker, script, and test folders exist. |
| Useful docs | Markdown contains concrete decisions, rules, inputs, outputs, and acceptance criteria. |
| MVP boundary | Bulk sheet export is MVP; Amazon Ads API execution is later. |
| Safety | Human approval requirement is documented across relevant areas. |

## Product Acceptance
- Customer can create product profile, upload research, review mappings, approve keywords, generate campaign plan, approve export, monitor 14 days, and review recommendations.
- No live ad changes are made in MVP.
- Every customer-impacting decision is auditable.
