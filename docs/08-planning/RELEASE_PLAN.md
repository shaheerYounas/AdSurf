# Release Plan

## Releases
| Release | Scope |
| --- | --- |
| R0 Documentation | Docs, folder scaffold, rules, architecture contracts. |
| R1 Internal MVP | Product profiles, uploads, parsing, scoring, keyword review. |
| R2 Campaign MVP | Campaign plan generation, negatives, bulk export approval. |
| R3 Monitoring MVP | Performance imports, 14-day rules, recommendations. |
| R4 Beta | Workspace roles, audit hardening, E2E tests, deployment pipeline. |
| R5 Later API | Amazon Ads API integration with approval-controlled execution. |

## Release Acceptance
Each release must include docs, tests, security review, and approval-flow verification for changed behavior.
